Subway Data Analysis¶

Introduction¶

O sistema de nibus e trens de Nova Iorque - o Metro Transit Authority - fornece seus dados para download atravs de arquivos CSV. Dentre as informaes disponveis esto os registros semanais de dados das catracas do metr.

Estes registros contm contagens cumulativas das entradas e sadas, normalmente agrupadas em perodos de 4 horas, com dados adicionais que permitem identificar a estao e catraca especfica correspondente a cada linha do arquivo. Neste projeto iremos utilizar um desses registros, mas no precisa baixar nada agora! O primeiro exerccio ser escrever um cdigo Python para fazer isso por voc :-)

Sobre este projeto¶

Neste projeto voc ir aplicar todos os conhecimentos adquiridos neste primeiro ms de curso, com tarefas bsicas de aquisio e limpeza de dados. No processo iremos descobrir informaes essenciais sobre os dados, utilizando o que foi aprendido no curso de estatstica.

O objetivo deste projeto explorar a relao entre os dados das catracas do metr de Nova Iorque e o clima no dia da coleta. Para isso, alm dos dados do metr, precisaremos dos dados de clima da cidade de Nova Iorque.

Os principais pontos que sero verificados neste trabalho:

  • Coleta de dados da internet
  • Utilizao de estatstica para anlise de dados
  • Manipulao de dados e criao de grficos simples com o Pandas

Sobre o dataset¶

Descrio das colunas

C/A,UNIT,SCP,STATION,LINENAME,DIVISION,DATE,TIME,DESC,ENTRIES,EXITS

C/A      = Agrupamento de catracas de que a catraca faz parte (_Control Area_)
UNIT     = Cabine de controle associada  estao onde a catraca se encontra (_Remote Unit for a station_)
SCP      = Endereo especfico da catraca (_Subunit Channel Position_)
STATION  = Nome da estao onde a catraca se encontra
LINENAME = Cdigo representando todas linhas que passam na estao*
DIVISION = Cdigo representando a concessionria original da linha, antes da prefeitura assumir a gesto   
DATE     = Representa a data (no formato MM-DD-YY) do evento de auditoria agendado
TIME     = Representa o horrio (hh:mm:ss) do evento de auditoria agendado
DESc     = Descreve o tipo de evento de auditoria registrado:
           1. "REGULAR" representando um evento de auditoria padro, em que a contagem  feita a cada 4 horas
           2. "RECOVR AUD" significa que o valor especfico estava perdido, mas foi recuperado posteriormente 
           3. Diversos cdigos sinalizam situaes em que auditorias so mais frequentes devido a atividades de
              planejamento ou soluo de problemas. 
ENTRIES  = A contagem cumulativa de entradas associadas  catraca desde o ltimo registro
EXITS    = A contagem cumulativa de sadas associadas  catraca desde o ltimo registro

*  Normalmente as linhas so representadas por um caractere. LINENAME 456NQR significa que os trens 4, 5, 6, N, Q e R passam pela estao.

def reducer():
    totalEntries = 0

    for line in sys.stdin:
        data = line.strip().split("\t")
        if len(data) != 2:
            continue
        
        thisKey, thisEntry = data
        
        try:
            thisEntry = float(thisEntry)

                print "{0}\t{1}".format(thisKey, totalEntries)

                totalEntries = 0
            
            oldKey = thisKey
